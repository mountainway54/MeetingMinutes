# 情書桌遊前端專案會議記錄

## 會議主題

建立一個以《情書》桌遊玩法為核心的卡牌對戰前端網站，第一階段先完成 2 人玩家對戰系統，之後再擴充登入、房間、排行榜、戰績等功能。

## 專案目標

本專案目標是製作一個可在瀏覽器中遊玩的卡牌對戰網站。玩家可以進入對戰畫面，依照情書桌遊規則輪流抽牌、出牌、指定對手、執行卡牌效果，直到其中一方勝利。

第一階段重點放在完整遊戲流程與前端互動體驗，不先追求大型平台功能。

## MVP 功能範圍

本階段先完成以下功能：

1. 2 人對戰模式
2. 遊戲初始化與洗牌
3. 16 張經典牌組規則
4. 玩家手牌、牌庫、棄牌區顯示
5. 回合流程控制
6. 每張卡牌效果判定
7. 玩家淘汰與勝負判定
8. 遊戲事件紀錄
9. 結束畫面與再來一局

暫不納入第一階段：

1. 登入註冊
2. 排行榜
3. 商店或卡包系統
4. 觀戰模式
5. 聊天系統
6. 多人房間大廳
7. 自訂卡牌或擴充牌組

## 五人分工規劃

### 成員 A：遊戲流程與狀態管理

負責整個遊戲的核心流程，確保遊戲可以從開始到結束順利進行。

主要功能：

1. 遊戲初始化
2. 洗牌與發牌
3. 移除隱藏牌與公開牌
4. 目前回合玩家判定
5. 抽牌、出牌、結束回合流程
6. 遊戲階段狀態管理
7. 與其他模組整合

建議負責檔案或模組：

1. `gameStore`
2. `gameReducer`
3. `turnManager`
4. `gameInitializer`

### 成員 B：卡牌規則與效果判定

負責實作每張牌的效果，以及特殊規則判定。

主要功能：

1. 守衛：猜牌淘汰
2. 牧師：查看對手手牌
3. 男爵：比較手牌點數
4. 侍女：保護狀態
5. 王子：指定玩家棄牌並重抽
6. 國王：交換手牌
7. 伯爵夫人：強制出牌規則
8. 公主：棄掉或打出即淘汰
9. 牌效執行後的結果回傳
10. 不合法操作阻擋

建議負責檔案或模組：

1. `cardEffects`
2. `ruleEngine`
3. `targetValidator`
4. `winConditionChecker`

### 成員 C：遊戲桌 UI 與卡牌互動

負責主要對戰畫面，讓玩家可以清楚看到牌桌狀態並進行操作。

主要功能：

1. 玩家資訊區
2. 對手資訊區
3. 我的手牌區
4. 牌庫顯示
5. 棄牌區顯示
6. 移除牌區顯示
7. 卡牌元件設計
8. 出牌互動
9. 選擇目標 UI
10. 猜牌選擇器 UI

建議負責檔案或模組：

1. `GameBoard`
2. `PlayerArea`
3. `OpponentArea`
4. `Card`
5. `DiscardPile`
6. `ActionPanel`

### 成員 D：頁面流程、結果頁與事件紀錄

負責遊戲外層頁面與輔助資訊，讓整個網站有完整使用流程。

主要功能：

1. 首頁或開始畫面
2. 玩家名稱輸入
3. 開始遊戲流程
4. 遊戲事件紀錄
5. 操作提示訊息
6. 淘汰原因顯示
7. 勝負結果頁
8. 再來一局
9. 錯誤與空狀態畫面

建議負責檔案或模組：

1. `HomePage`
2. `StartPanel`
3. `GameLog`
4. `ResultModal`
5. `Toast`
6. `RoundSummary`

### 成員 E：視覺設計、動畫與響應式體驗

負責整體視覺風格、卡牌質感、動畫效果與不同裝置上的體驗。

主要功能：

1. 網站視覺風格設計
2. 卡牌版面設計
3. 牌桌背景與布局
4. 出牌動畫
5. 抽牌動畫
6. 淘汰與勝利動畫
7. RWD 手機版與桌機版
8. 深色或主題色設計
9. 按鈕、彈窗、狀態標籤樣式
10. 基礎可用性與易讀性檢查

建議負責檔案或模組：

1. `theme`
2. `layout`
3. `animations`
4. `responsiveStyles`
5. `CardVisual`

## 建議技術分工

| 項目 | 建議技術 |
|---|---|
| 前端框架 | React 或 Next.js |
| 語言 | TypeScript |
| 狀態管理 | Zustand 或 Redux Toolkit |
| 動畫 | Framer Motion |
| 樣式 | Tailwind CSS 或 CSS Modules |
| 測試 | Vitest / React Testing Library |
| 部署 | Vercel |

第一階段可先做本地 2 人對戰，也就是同一台電腦或同一個瀏覽器輪流操作。等核心規則穩定後，再加入 Socket.IO 做線上房間同步。

## 四週開發時程

### 第 1 週：規格確認與基礎架構

目標：

1. 確認完整遊戲規則
2. 建立專案架構
3. 建立卡牌資料結構
4. 完成基本頁面切版
5. 完成遊戲初始化流程

交付成果：

1. 可進入遊戲畫面
2. 可產生牌組、洗牌、發牌
3. 可顯示玩家、手牌、牌庫與棄牌區

### 第 2 週：回合流程與卡牌效果

目標：

1. 完成抽牌與出牌流程
2. 完成 8 種卡牌效果
3. 完成合法操作判定
4. 完成事件紀錄

交付成果：

1. 玩家可輪流抽牌與出牌
2. 所有卡牌效果可正常執行
3. 遊戲紀錄能顯示主要事件

### 第 3 週：勝負判定、UI 優化與動畫

目標：

1. 完成淘汰判定
2. 完成牌庫空時比點數
3. 完成結果畫面
4. 加入主要動畫與操作提示
5. 優化手機與桌機版畫面

交付成果：

1. 遊戲可完整玩完一局
2. 可顯示勝者與淘汰原因
3. 操作體驗接近正式 Demo

### 第 4 週：整合測試與展示準備

目標：

1. 修正規則錯誤
2. 修正 UI 問題
3. 測試各種勝負情境
4. 優化動畫與載入狀態
5. 準備展示流程與簡報素材

交付成果：

1. 穩定可展示版本
2. 完整 Demo 流程
3. 專案 README
4. 分工與技術說明
